# 学科知识整合智能体项目交付总报告

生成日期：2026-05-10

公开仓库：https://github.com/Anchorite-no/ai-textbook-kg-agent

## 1. 项目定位

本项目面向“学科知识整合智能体”赛题，目标是把多本教材和常见办公文档解析为统一证据链，构建多层知识图谱，并支持 RAG / GraphRAG 问答、跨教材去重压缩、教师通过对话修正整合决策。

核心证据链：

```text
RawFile -> DocumentElement -> Section -> Chunk -> KnowledgeNode / KnowledgeEdge -> RAG Evidence -> IntegrationDecision -> TeacherEdit
```

## 4. 核心能力

### 多格式解析

已支持：

- `txt`
- `md` / `markdown`
- `pdf`
- `docx`
- `xlsx`
- `csv` / `tsv`
- `pptx`

旧版 `doc/xls/ppt` 暂不直接解析，接口会返回明确错误，后续可通过 LibreOffice 转换补齐。

### KG 构建

当前 KG builder 使用 `kg_extract_v4_context_links`：

- 支持 LLM 抽取，未配置 LLM 时使用确定性 fallback。
- 所有节点和边都有稳定 ID。
- 所有节点和边保留 `source_locator` 与 `evidence_chunk_ids`。
- 增加上下文弱链接：同章节、同 chunk、相邻章节。
- 弱链接通过 `metadata.contextual_edge=true`、`metadata.extraction_method=context_link` 标记，供展示与复核，不直接作为强合并证据。

### RAG / GraphRAG

- RAG 使用本地 BM25 + hash embedding 混合检索。
- GraphRAG 联合 chunk、node hit、path、integration decision。
- GraphRAG 状态已覆盖 7 本书、16,585 节点、45,800 边。

## 7. 风险与后续

- 前端展示层仍需继续打磨视觉密度和大图布局。
- 章节树暂时隐藏，后续应接入更强清洗后的教学目录结构。
- LLM 全量抽取没有默认开启，当前 demo 为确定性 fallback + 上下文弱链接；可对关键教材开启 LLM 进行质量增强。
- GraphRAG 当前已可演示，后续可接更强 embedding / vector store 提升召回质量。
