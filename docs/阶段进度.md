## 计划 02 已完成内容

- `converted_textbooks`：保留为教材演示导入入口，并已做最小正文结构清洗。
- `txt/md/markdown`：上传后进入统一解析层，按标题或固定窗口切分，保留行号。
- `pdf`：逐页抽取文本，过滤高频页眉页脚；优先识别章节，失败时按 5 页窗口切分。
- `docx`：按 Heading 分 section，表格按原位置转 Markdown 表格，保留段落范围。
- `xlsx/csv/tsv`：识别表头，跳过空行列，按 40 行窗口切分，保留 sheet/行范围。
- `pptx`：每页 slide 一个 section，按 top/left 合并文本框，抽取备注和表格。
- `.doc/.xls/.ppt`：暂不直接解析，返回清晰错误；后续通过 LibreOffice 转换兜底。
- `/api/textbooks/upload`：兼容单文件上传和无文件 demo 导入。
- `/api/textbooks/upload-batch`：批量上传，单文件失败不会阻断其它文件。
- `/api/textbooks/{raw_file_id}/parse`：对已保存上传文件重新解析。
- `/api/uploads/sessions`：大文件分片上传会话，支持断点式补传、sha256 合并校验和完成后解析。
- `/api/textbooks/upload-async`：上传后立即返回异步解析 job。
- `/api/textbooks/{raw_file_id}/parse-async`：对已保存上传文件启动后台重解析。
- `/api/uploads/sessions/{session_id}/complete-async`：大文件后台合并并解析。
- `/api/jobs/{job_id}/retry`：重试带上下文的失败异步任务。
- `/api/rag/index`：建立 chunk 证据索引。
- `/api/rag/status`：查看索引状态。
- `/api/rag/query`：返回带 citations 和 source_chunks 的证据问答。
- `backend/scripts/smoke_phase2.py`：覆盖解析器和 API 烟测。
- `backend/scripts/smoke_00_stage3_async.py`：覆盖异步 DAG、失败重试和分片 complete-async。
- `backend/scripts/smoke_00_stage4_rag.py`：覆盖索引、查询、引用和无答案拒答。
- `backend/scripts/export_openapi.py`：导出 `docs/openapi.snapshot.json` 供前端 codegen。
- 全局错误响应已统一为 `{ message, code, detail }`。

## 00 阶段 8 已完成内容

- `POST /api/integration/build`：基于阶段 7 的术语对齐和阶段 5/6 的 KG 生成跨教材整合结果。
- `GET /api/integration?raw_file_ids={id1},{id2}`：读取整合与压缩结果。
- `IntegrationDecision`：输出 `merge`、`keep`、`remove`、`refine`、`conflict`，每条决策携带 `reason`、`confidence`、`evidence_chunk_ids`、`source_locators`。
- `IntegratedConcept`：提供整合后概念视图，但不覆盖原始 `KnowledgeNode`。
- `CompressionStats`：记录原始 section 字符数、保留字符数、目标压缩率、实际压缩率、节点变化和证据完整度。
- 去重与压缩分离：`merge/refine/conflict` 判断概念关系；`remove` 只把低重要度 singleton 从整合正文移出，不删除原始证据。
- 冲突说法单独标记为 `conflict`，并强制保留双方原始说法等待教师裁决。
- `backend/scripts/smoke_00_stage8_integration.py`：覆盖五类决策、缓存读取、压缩率和证据链。
- `backend/scripts/benchmark_00_stage8_integration.py`：覆盖五类动作、压缩率 <= 30%、证据完整度、节点数量变化。

阶段 8 当前仍不处理：

- 教师对 merge/remove/refine/conflict 的确认、驳回和局部重算。
- GraphRAG 路径检索。
- 对整合结果的前端编辑交互。

## 00 阶段 9 已完成内容

- `GET /api/graphrag/status`：查看 RAG index、Graph、Alignment、Integration 是否可用于 GraphRAG。
- `POST /api/graphrag/query`：返回带引用的 `answer`、`citations`、`source_chunks`、`node_hits`、`paths`、`decisions`。
- 意图识别覆盖：定义、出现在哪些教材、跨教材差异、前置知识、两个概念关系、整合决策解释、混合问答。
- 查询链路：术语识别、alias 扩展、chunk 检索、KnowledgeNode 检索、图邻居/路径扩展、IntegrationDecision 查询、证据 rerank。
- 路径输出：`GraphRagPath` / `GraphRagPathStep` 明确展示节点、边、关系类型、证据 chunk 和 `source_locator`。
- 拒答边界：没有 chunk 命中且没有精确术语/别名命中时拒答；混合关系问题如果只命中一个局部术语且 query coverage 不足，也拒答。
- `backend/scripts/smoke_00_stage9_graphrag.py`：覆盖 6 类可答问题和无答案拒答。
- `backend/scripts/benchmark_00_stage9_graphrag.py`：覆盖 citation、node、path、decision、无答案拒答指标。
