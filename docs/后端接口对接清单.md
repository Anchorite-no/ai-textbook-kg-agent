# 后端接口对接清单

面向前端联调使用。类型以 `docs/openapi.snapshot.json` 为准；本文只列工作台需要的主路径和注意事项。

## 基础

| 方法 | 路径 | 用途 | 主要响应 |
| --- | --- | --- | --- |
| GET | `/api/health` | 健康检查 | `HealthResponse` |
| GET | `/api/jobs/{job_id}` | 查询任务状态 | `JobRecord` |
| POST | `/api/jobs/{job_id}/retry` | 重试失败任务 | `JobRetryResponse` |

## 教师编辑与对话

| 方法 | 路径 | 用途 | 主要响应 |
| --- | --- | --- | --- |
| GET | `/api/teacher-edits?raw_file_ids=a,b` | 教师修改事件列表 | `TeacherEditListResponse` |
| POST | `/api/teacher-edits` | 记录通用教师修改事件 | `TeacherEditApplyResponse` |
| GET | `/api/dialogue/messages?raw_file_ids=a,b` | 对话历史 | `DialogueHistoryResponse` |
| POST | `/api/dialogue/messages` | 发送教师对话消息 | `DialogueMessageResponse` |

直接覆盖决策示例：

```json
POST /api/integration/decisions/integration_decision_xxx/override
{
  "raw_file_ids": ["raw_a", "raw_b"],
  "action": "conflict",
  "retained_content": "教师要求保留两种说法，课堂上单独说明差异。",
  "reason": "该合并存在教学风险，需要复核。",
  "confidence": 1.0,
  "created_by": "teacher"
}
```

对话修改决策示例：

```json
POST /api/dialogue/messages
{
  "raw_file_ids": ["raw_a", "raw_b"],
  "message": "请把 integration_decision_xxx 改为保留，避免误删课堂重点。",
  "created_by": "teacher",
  "retained_content": "教师要求保留该知识点。"
}
```

## 前端接入建议

- 所有接口错误统一为 `{ message, code, detail }`。
- `source_locator` 是证据展示的核心字段，不要在 adapter 层丢掉。
- 图谱首屏建议 `top_n=200`，节点详情再按需调用 `/api/graph/nodes/{node_id}`。
- 整合决策列表以 `IntegrationResponse.decisions` 为数据源，压缩率来自 `compression_stats.compression_ratio`。
- 教师覆盖后使用返回的 `integration` 立即刷新右侧整合面板；同时追加 `edit` 到本地事件列表。
- 对话 tab 使用 `DialogueMessageResponse.user_message` 和 `assistant_message` 立即渲染，不必等再次拉取历史。
