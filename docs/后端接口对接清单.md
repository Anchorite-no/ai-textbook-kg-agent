# 后端接口对接清单

面向前端联调使用。类型以 `docs/openapi.snapshot.json` 为准；本文只列工作台需要的主路径和注意事项。

## 基础

| 方法 | 路径 | 用途 | 主要响应 |
| --- | --- | --- | --- |
| GET | `/api/health` | 健康检查 | `HealthResponse` |
| GET | `/api/jobs/{job_id}` | 查询任务状态 | `JobRecord` |
| POST | `/api/jobs/{job_id}/retry` | 重试失败任务 | `JobRetryResponse` |

## 七本书数据集与一键整理

七本书 demo 不是单独假数据，而是当前后端正常数据目录里的真实产物：`data/parsed`、`data/graphs`、`data/indexes/rag_index.json`、`data/alignments`、`data/integrations`。前端先读数据集状态拿到 `raw_file_ids`，后续仍调用教材、图谱、RAG、对齐、整合、GraphRAG 的正式接口。

| 方法 | 路径 | 用途 | 主要响应 |
| --- | --- | --- | --- |
| GET | `/api/datasets/seven-books` | 读取七本教材全量数据集状态、指标和正式接口入口 | `SampleDatasetResponse` |
| POST | `/api/datasets/seven-books/prepare` | 从 `materials/converted_textbooks` 重新导入七本书并生成正式后端产物 | `SampleDatasetPrepareResponse` |
| POST | `/api/workflows/organize` | 前端上传一个或多个文件后，一键解析并生成 KG/RAG/对齐/整合 | `OrganizeWorkflowAcceptedResponse` |

`POST /api/workflows/organize` 使用 `multipart/form-data`：

- `files`：一个或多个文件。
- `build_graph/build_layered_graphs/build_rag/build_alignment_graph/build_integration_result`：默认 `true`。
- `use_llm`：默认 `false`，需要 LLM 抽取时显式传 `true`。
- 其它参数：`max_sections`、`max_nodes_per_section`、`alignment_min_confidence`、`alignment_max_nodes`、`integration_target_compression_ratio`、`integration_max_nodes`。

该接口立即返回 `job.id`；任务完成后，`GET /api/jobs/{job_id}` 的 `result.raw_file_ids` 和 `result.endpoints` 就是前端继续读取正式数据的入口。

## 教师编辑与对话

| 方法 | 路径 | 用途 | 主要响应 |
| --- | --- | --- | --- |
| GET | `/api/teacher-edits?raw_file_ids=a,b` | 教师修改事件列表 | `TeacherEditListResponse` |
| POST | `/api/teacher-edits` | 记录通用教师修改事件 | `TeacherEditApplyResponse` |
| GET | `/api/dialogue/messages?raw_file_ids=a,b` | 对话历史 | `DialogueHistoryResponse` |
| POST | `/api/dialogue/messages` | 发送教师对话消息 | `DialogueMessageResponse` |

直接覆盖决策示例：

```json
POST /api/integration/decisions/integration_decision_xxx/override
{
  "raw_file_ids": ["raw_a", "raw_b"],
  "action": "conflict",
  "retained_content": "教师要求保留两种说法，课堂上单独说明差异。",
  "reason": "该合并存在教学风险，需要复核。",
  "confidence": 1.0,
  "created_by": "teacher"
}
```

对话修改决策示例：

```json
POST /api/dialogue/messages
{
  "raw_file_ids": ["raw_a", "raw_b"],
  "message": "请把 integration_decision_xxx 改为保留，避免误删课堂重点。",
  "created_by": "teacher",
  "retained_content": "教师要求保留该知识点。"
}
```

## 前端接入建议

- 所有接口错误统一为 `{ message, code, detail }`。
- `source_locator` 是证据展示的核心字段，不要在 adapter 层丢掉。
- 图谱首屏建议 `top_n=1000`，节点详情再按需调用 `/api/graph/nodes/{node_id}`；前端可继续通过布局筛选避免一次性显示过密。
- 七本书首页先调用 `/api/datasets/seven-books`；不要维护另一份 demo fixture。
- 整合决策列表以 `IntegrationResponse.decisions` 为数据源，压缩率来自 `compression_stats.compression_ratio`。
- 教师覆盖后使用返回的 `integration` 立即刷新右侧整合面板；同时追加 `edit` 到本地事件列表。
- 对话 tab 使用 `DialogueMessageResponse.user_message` 和 `assistant_message` 立即渲染，不必等再次拉取历史。
