## KG 质量评估

本轮重点修复“节点少、边少、零散节点过多”的问题。

改动前：

- 每本默认只跑 120 个 section。
- 每节默认 8 个节点。
- 关系要求偏强，跨章节/同文档上下文不补边。
- 孤立节点率约 39.7%。

改动后：

- 七本书覆盖 2,014 / 2,014 个 section。
- 默认 12 节点/section，支持请求上限到 80。
- 图构建版本：`kg_extract_v4_context_links`。
- 新增 28,395 条上下文弱链接边。
- 孤立节点率降至 3.64%。

## 证据链评估

满足要求：

- `Chunk.source_locator` 覆盖率 100%。
- `KnowledgeNode.source_locator` 覆盖。
- `KnowledgeEdge.source_locator` 覆盖。
- RAG citation 返回 `source_locator` 和 quote。
- GraphRAG path step 返回 `source_locator` 和 evidence chunk。
- IntegrationDecision 保留 `source_locators` 和 `evidence_chunk_ids`。
- TeacherEdit 不覆盖原始证据，只追加事件。

## 薄弱项与建议

1. LLM 抽取质量：当前全量 demo 默认未启用 LLM，建议赛后对高价值章节开启 LLM 增强。
2. 章节清洗：后端已有 section/chunk，但教材真实目录结构仍需进一步清洗，前端已把章节树限制在固定滚动区域展示。
3. 语义合并：当前对齐以本地相似度为主，后续可加入医学 embedding 与 LLM judge。
4. 大图展示：后端已提高 `top_n=1000`，前端仍需分页、聚类和层级筛选。

## 结论

后端已经满足黑客松演示的核心要求：多格式解析、统一 JSON、证据链、KG、RAG、GraphRAG、跨教材整合、教师反馈闭环和报告生成均已跑通。当前最值得继续投入的是抽取质量和前端可视化，而不是再补新的后端主链路。
