## 推荐演示流程

1. 打开前端首页，展示工作台。
2. 点击顶部“载入示例”，加载七本医学教材数据集。
3. 展示顶部指标：教材数、chunk 数、任务状态。
4. 切换图谱模式：
   - 单本：展示当前教材图谱。
   - 并列：展示多本书拼接图谱。
   - 对照：用于跨书对比浏览。
5. 在搜索框输入医学概念，例如“炎症”“血液循环”“细胞损伤”。
6. 点击节点，展示节点详情、置信度、证据 chunk、来源定位。
7. 切到 RAG / GraphRAG 面板，提问：
   - “炎症反应的基本过程是什么？”
   - “细胞损伤和细胞死亡有什么关系？”
   - “血液循环相关知识在哪些教材中出现？”
8. 切到整合面板，展示压缩率、合并/保留/细化/冲突决策。
9. 对某个整合决策点击“保留 / 冲突 / 合并 / 移除 / 细化”，展示 TeacherEdit 记录与重新整合效果。
10. 切到对话面板，输入教师反馈，例如：
    - “这个决策不要合并，课堂上需要保留两个概念的差异。”
11. 点击顶部“生成报告”，展示后端 `/api/report/generate` 生成的 Markdown 报告。
12. 点击左侧“添加”，上传一个或多个文件，演示从前端触发完整整理流程。

## 关键讲解点

- demo 数据不是假 fixture，七本书和后续上传文件共用同一套正式接口。
- 所有节点、边、chunk、整合决策都可回溯到 `source_locator`。
- KG 使用强语义边 + 上下文弱链接，解决节点过少和零散节点过多问题。
- RAG 回答带 chunk 引用，GraphRAG 额外返回节点命中、路径和整合决策。
- 教师修改不会覆盖原始数据，而是追加 `TeacherEdit` 事件，保证可审计。

## 应急方案

- 如果图谱太密，使用搜索框聚焦概念，或切回“单本”。
- 如果章节结构被问到，说明当前演示阶段已隐藏左侧章节树，后端仍保留统一 JSON 与 source locator，后续可接更强目录清洗。
- 如果 LLM 调用不稳定，说明 demo 默认使用确定性 fallback，LLM 可作为增强模式开启。
