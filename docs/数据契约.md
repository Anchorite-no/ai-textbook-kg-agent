# 数据契约

本项目所有解析、知识抽取、去重压缩、RAG 和教师编辑都围绕同一条证据链展开：

```text
RawFile -> DocumentElement -> Section -> Chunk -> KnowledgeNode / KnowledgeEdge -> Evidence / Decision / TeacherEdit
```

## 通用约束

- 每个对象必须有稳定 `id`。当前实现用来源文件 hash、页码、section 页范围、chunk 字符范围生成。
- `Chunk`、`KnowledgeNode`、`KnowledgeEdge`、`RetrievalEvidence` 必须直接携带 `source_locator`。
- 自动合并、删除、压缩只能通过 `IntegrationDecision` 记录，不能覆盖原始证据。
- 教师修改通过 `TeacherEdit` 记录为事件，后续再触发局部再整合。

## SourceLocator

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `raw_file_id` | string | 来源文件 ID |
| `source_path` | string | 原始文件或转换目录路径 |
| `source_type` | string | `uploaded` / `converted_textbook` / `generated` |
| `locator_text` | string | 人类可读定位，例如 `生理学 pages 10-11` |
| `page_start/page_end` | number/null | PDF 或教材页码范围 |
| `line_start/line_end` | number/null | TXT/MD 行范围 |
| `sheet_name,row_start,row_end` | string/number/null | Excel 定位 |
| `slide_number` | number/null | PPT 定位 |
| `char_start/char_end` | number/null | 当前 section 内字符范围 |
| `element_ids` | string[] | 关联的 DocumentElement |
| `quote_hash` | string/null | 证据文本 hash，用于检测漂移 |

## RawFile

表示原始输入文件或已转换教材。

核心字段：`id`、`original_filename`、`title`、`format`、`source_type`、`storage_path`、`sha256`、`size_bytes`、`page_count`、`text_char_count`、`metadata`。

## DocumentElement

清洗后的最小文档元素。当前 converted_textbooks 导入器先按页生成 `page` element。

核心字段：`id`、`raw_file_id`、`type`、`text`、`order_index`、`parent_section_id`、`source_locator`、`char_count`、`metadata`。

可选类型：`heading`、`paragraph`、`table`、`figure`、`list`、`page`、`slide`、`sheet`、`note`。

## Section

面向解析和抽取的结构单元。当前第一阶段按 5 页窗口生成 `page_window` section，后续会升级为章节、slide、sheet 等真实结构。

核心字段：`id`、`raw_file_id`、`title`、`section_type`、`level`、`order_index`、`parent_section_id`、`element_ids`、`content`、`char_count`、`source_locator`、`metadata`。

## Chunk

RAG 和证据引用的基本单元。当前默认约 700 中文字符，80 字 overlap。

核心字段：`id`、`raw_file_id`、`section_id`、`text`、`order_index`、`char_start`、`char_end`、`char_count`、`source_locator`、`metadata`。

每个 chunk 的 `source_locator` 必须能回到页码、section 字符范围和关联 element。

## KnowledgeNode

知识点节点。

核心字段：`id`、`name`、`node_type`、`definition`、`aliases`、`source_locator`、`evidence_chunk_ids`、`confidence`、`metadata`。

节点类型：`Concept`、`Term`、`Mechanism`、`Process`、`Structure`、`Function`、`Disease`、`Symptom`、`Pathogen`、`Diagnosis`、`Treatment`、`Experiment`、`Example`。

## KnowledgeEdge

知识关系边。

核心字段：`id`、`source_node_id`、`target_node_id`、`relation_type`、`description`、`source_locator`、`evidence_chunk_ids`、`confidence`、`metadata`。

关系类型：`ALIAS_OF`、`SAME_AS`、`IS_A`、`PART_OF`、`PREREQUISITE_OF`、`CAUSES`、`LEADS_TO`、`APPLIES_TO`、`CONTRASTS_WITH`、`EXPLAINS`、`EVIDENCED_BY`、`MENTIONED_IN`、`REFINES`、`CONFLICTS_WITH`。

## ConceptCluster

跨教材术语对齐和去重结果。

核心字段：`id`、`canonical_name`、`aliases`、`member_node_ids`、`evidence_chunk_ids`、`confidence`、`metadata`。

## IntegrationDecision

压缩和整合决策。

核心字段：`id`、`cluster_id`、`action`、`target_node_ids`、`retained_content`、`removed_redundancy`、`reason`、`confidence`、`evidence_chunk_ids`、`created_at`、`metadata`。

动作类型：`merge`、`keep`、`remove`、`refine`、`conflict`。

## TeacherEdit

教师修改事件。

核心字段：`id`、`target_type`、`target_id`、`operation`、`before`、`after`、`reason`、`created_by`、`created_at`、`affected_ids`、`metadata`。

操作类型：`create_node`、`update_node`、`delete_node`、`split_node`、`merge_nodes`、`create_edge`、`update_edge`、`override_decision`。

## RetrievalEvidence

RAG / GraphRAG 检索证据。

核心字段：`id`、`query`、`chunk_id`、`raw_file_id`、`source_locator`、`relevance_score`、`answer_span`、`metadata`。

## 第一阶段导入输出

调用：

```text
POST /api/textbooks/upload
```

若不上传文件，接口会从 `materials/converted_textbooks` 导入第一本教材，生成：

```text
data/parsed/{raw_file_id}.json
```

JSON 顶层结构：

```json
{
  "id": "parsed_xxx",
  "raw_file": {},
  "elements": [],
  "sections": [],
  "chunks": [],
  "generated_at": "2026-05-10T...",
  "metadata": {
    "source_pipeline": "materials.converted_textbooks",
    "chunk_size": 700,
    "chunk_overlap": 80
  }
}
```

当前 converted_textbooks 导入器已加入最小结构清洗：

- 跳过封面、版权、编委、前言、目录等正文前噪声页。
- 识别 `绪论`、`第X章`、`第X节`、`一、二、三、` 等标题。
- 删除重复页眉、纯页码、数字资源提示等展示噪声。
- 保留原始页级 `DocumentElement`，清洗后的 `Section` 和 `Chunk` 仍通过 `source_locator.page_start/page_end` 追溯回原文页。

辅助接口：

| 方法 | 路径 | 作用 |
| --- | --- | --- |
| `GET` | `/api/textbooks` | 查看已解析教材列表 |
| `GET` | `/api/textbooks/{raw_file_id}` | 查看某本教材的统一 JSON |
